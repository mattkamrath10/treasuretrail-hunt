PHASE 8 — PERFORMANCE, STABILITY, AND DEPLOYMENT HARDENING

# GOAL

TreasureTrail is now a large, state-heavy, feed-driven application.

Before adding major new systems, harden:

* performance
* stability
* deployment behavior
* caching
* mobile responsiveness
* rendering efficiency

This phase should make the app feel:

* faster
* smoother
* more reliable
* more production-ready

DO NOT:

* add large new features
* add AI systems
* add complex new backend architecture
* add gamification

Focus ONLY on hardening and optimization.

1. GLOBAL PERFORMANCE AUDIT
   ========================================

Audit:

* rerenders
* feed updates
* image rendering
* modal rendering
* memory usage
* polling behavior
* route transitions
* state updates

Identify:

* expensive renders
* duplicate fetches
* unnecessary state churn
* layout thrashing
