PART 2 — FIX BROKEN PROFILE SHARE LINKS

Current bug:
When sharing a profile from mobile, the generated URL downloads a .txt file (example: mattkamrath.txt) instead of opening a real public profile page.

Fix requirements:
1. Create a public shareable profile route:
   /profile/:username
   OR
   /u/:username

2. Share links should generate URLs like:
   https://treasuretrail-hunt.com/profile/mattkamrath

3. Public profile pages must work WITHOUT login

4. Public profile page should display:
   - username
   - avatar
   - finds
   - hosted live events
   - wanted posts
   - follower count

5. If profile does not exist:
   show branded "Profile not found" page

6. Share button behavior:
   - use navigator.share on mobile when available
   - fallback copies URL to clipboard

7. Remove ALL .txt generation/download behavior entirely

8. Do NOT break existing logged-in profile behavior
