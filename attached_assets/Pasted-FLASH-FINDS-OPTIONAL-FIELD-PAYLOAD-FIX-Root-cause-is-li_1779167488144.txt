FLASH FINDS OPTIONAL-FIELD PAYLOAD FIX

Root cause is likely malformed payload merging when optional upload fields are selected.

Current evidence:

* simple uploads sometimes work
* uploads with additional metadata often break
* blank cards appear
* cards collapse
* image disappears
* only “Found” badge renders
* renderer receives inconsistent object shapes

This is now a payload normalization + optimistic merge issue.

GOAL

Every Flash Find upload must generate ONE guaranteed-valid canonical object shape before:

DB insert
optimistic prepend
feed render

DO NOT pass raw form state directly into feed state.

IMPLEMENT:

createCanonicalFlashFindPayload(formState)

Normalize ALL optional fields before insert.

Required final shape:

{
title: string,
caption: string,
description: string,
category: string,
location_found: string | null,
marketplace_found: string | null,
price_estimate: number | null,
scout_needed: boolean,
image_url: string | null,
created_at: string,
user_id: string
}

CRITICAL RULES

If optional field is object/array:

* convert to string before insert

Never allow:
undefined
nested objects
empty arrays
NaN
invalid URLs

Before optimistic prepend:

normalize payload AGAIN.

Then:

validateFeedItem(payload)

If invalid:

* abort prepend
* log exact field failure
* surface toast
* DO NOT render malformed card
